\# Coding Test – AI Product Requirement \& UI



\# Generator



\## Objective



Build an application that converts a meeting transcript into:



\- Product Requirements Document (PRD)

\- User Stories

\- Functional Requirements

\- UI/UX Design (Wireframes)



\## Time Limit



5 hours



\## Input



A meeting transcript provided as text or uploaded file.



\## Expected Outputs



1\. Product Requirements Document (PRD)

2\. User Stories

3\. Functional Requirements

4\. UI/UX Design (wireframes or screen designs)



\## PRD Structure



\- Product Overview

\- Problem Statement

\- Goals

\- User Personas

\- User Journey

\- Features

\- Functional Requirements

\- Non-Functional Requirements

\- Success Metrics



\## User Stories



Generate user stories using the format:

As a \[user], I want \[action] so that \[benefit].



\## Functional Requirements



Generate a structured requirement list with IDs such as FR-001, FR-002, etc.



\## UI/UX Design





Generate:



\- Sitemap

\- Screen List

\- Low-fidelity Wireframes

\- User Flow



\## Technical Requirements



Frontend framework is flexible (React, Next.js, Vue, Svelte, etc.).

Any AI model or LLM may be used (My Oppinion: I think we use openai API services). 

Database/storage is optional.



\## Evaluation Criteria



AI Extraction Accuracy – 35%

Product Thinking – 20%

UI/UX Generation – 20%

Code Quality – 15%

UX \& Presentation – 10%
